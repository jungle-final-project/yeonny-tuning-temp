# Chrome Remote Desktop 원격 지원 수동 검증

## 목적과 경계

이 문서는 관리자가 승인한 AS 티켓에서 Chrome Remote Desktop으로 실제 지원을 연결하고, 서비스에 시작·완료 상태를 기록하는 수동 절차다. 서비스는 승인 상태, 일회용 코드 전달, 시작·완료 기록만 담당한다. Google 로그인, 코드 생성·입력, 화면 공유 승인, 실제 화면·입력 제어는 사람이 Chrome Remote Desktop 공식 페이지에서 직접 수행한다.

공식 지원 페이지: <https://remotedesktop.google.com/support>

## 사전 조건

- PC Agent 진단 결과와 원격 지원 AS 요청이 실제 데이터로 생성되어 있어야 한다.
- 관리자 검토에서 원격 지원이 승인되고 담당 관리자가 배정되어 있어야 한다.
- 사용자와 관리자는 서로 다른 로그인 세션을 사용한다.
- 일회용 지원 코드를 채팅, 로그, 이슈, 스크린샷에 남기지 않는다.

## 수동 검증 절차

1. PC Agent에서 실제 진단을 실행하고 사용자가 원격 지원 방식으로 AS를 요청한다.
2. 관리자가 AS 상세 상담방 아래의 `Chrome 원격지원 안내 보내기`를 실행한다. 기존 관리자 승인 트랜잭션으로 티켓이 원격 선점검 상태로 전환되고 공식 Chrome 지원 주소와 코드 등록 안내가 상담 메시지로 전송된다.
3. 사용자 AS 상세의 `원격 지원 연결`에서 `Chrome Remote Desktop 열기`를 눌러 공식 지원 페이지를 새 탭으로 연다.
4. 사용자가 Google 계정 로그인 등 공식 페이지 절차를 직접 완료하고 실제 일회용 지원 코드를 생성한다.
5. 사용자가 서비스의 지원 코드 입력란에 코드를 등록한다.
6. 관리자 AS 상세에서 상태가 `지원 코드 등록 완료`로 바뀌고 마스킹된 코드만 표시되는지 확인한다.
7. 담당 관리자가 `코드 복사`를 눌러 전체 코드를 클립보드로 복사한다. 화면 DOM과 일반 티켓 응답에 전체 코드가 상시 남지 않는지 확인한다.
8. 관리자가 `Chrome Remote Desktop 열기`로 공식 페이지를 새 탭에서 열고 복사한 코드를 직접 입력한다.
9. 사용자가 Chrome Remote Desktop 화면에서 공유 요청 내용을 확인하고 화면 공유를 직접 승인한다.
10. 관리자와 사용자가 실제 화면 표시와 허용된 키보드·마우스 입력 제어가 동작하는지 Chrome Remote Desktop에서 확인한다.
11. 실제 연결을 확인한 뒤에만 관리자가 서비스의 `지원 시작`을 눌러 `IN_PROGRESS`와 시작 시각을 기록한다.
12. 원격 점검을 마치고 Chrome Remote Desktop 연결을 사람이 직접 종료한다.
13. 관리자가 서비스에서 `지원 완료`를 눌러 `COMPLETED`와 완료 시각을 기록한다.
14. 사용자·관리자 화면을 새로고침해 완료 상태가 유지되고 코드 입력·교체·복사 버튼이 사라졌으며 DB의 일회용 코드가 제거되었는지 확인한다.

## 실패 시 확인

- 승인 전 사용자 화면에 연결 영역이 보이면 권한·노출 조건 오류다.
- 다른 사용자 또는 담당자가 아닌 관리자가 상태나 코드를 조회할 수 있으면 즉시 보안 결함으로 처리한다.
- 코드 없이 시작되거나 완료 후 전체 코드가 다시 조회되면 상태 전이 결함이다.
- Chrome Remote Desktop 연결 성공 여부는 서비스가 자동 감지하지 않는다. 관리자가 실제 연결을 확인하기 전 `지원 시작`을 누르지 않는다.

## 운영 주의

- 이번 구현은 비공개 Google API, 외부 페이지 DOM 조작, iframe, 브라우저 자동화, 자체 원격제어 프로토콜을 사용하지 않는다.
- Google 정책·UI·코드 형식은 변경될 수 있으므로 특정 자릿수에 의존하지 않는다.
- 현재 저장소에는 재사용 가능한 필드 암호화 유틸리티가 없어 코드 원문은 DB 접근 제한, 로그 비노출, 완료 후 삭제로 보호한다. 운영 배포 전 저장 시 암호화와 비밀정보 접근 감사를 별도로 설계한다.
