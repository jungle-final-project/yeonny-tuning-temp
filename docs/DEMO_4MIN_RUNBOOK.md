# 최종 4분 발표 시나리오 실행 가이드

이 문서는 메인 AI 검색에서 시작해 최소 게이밍 견적 추천, 목표 FPS 개선, 의도적인 호환 실패 복구, 견적 저장·비교, 예상 견적서 확인까지 한 사용자 흐름으로 재현하는 발표 기준이다. 발표 문장은 예시이며, 아래의 입력과 화면 불변식을 통과 기준으로 사용한다.

## 발표 전 준비

1. 최신 `main`을 배포한 뒤 `/api/health`가 `UP`인지 확인한다.
2. 공용 연습 계정 대신 발표 전용 사용자 계정을 사용한다. 다른 사용자의 연습 상태가 섞이지 않아야 한다.
3. 발표 사용자 계정에 비교용 고성능 저장 견적 1개가 있어야 한다.
4. AI 대화 기록과 현재 quote draft를 초기화한다. 브라우저 저장소만 비우지 말고 서버의 현재 draft가 비었는지도 확인한다.
5. `중학교 3학년 조카가 처음 PC를 맞추려고 합니다. 배틀그라운드가 잘 돌아가는 컴퓨터를 원합니다.` → `배그 같은 게임이 잘 돌아갔으면 좋겠어요.` → `최소 견적으로 맞춰줘.`를 미리 한 번 실행해 추천 카드가 정확히 3개인지 확인한다.
6. 첫 번째 최소 구성의 PUBG QHD 평균 FPS가 140FPS 미만인지 확인한다. 현재 demo seed에서는 RTX 5060의 QHD 평균 130FPS가 기준이다. 값이 바뀌면 발표용 초기 카드 또는 목표 FPS를 근거에 맞게 다시 고정한다.
7. 현재 구성에 적용했을 때 `size` Tool이 `FAIL`이 되는 케이스 1개를 미리 확인한다. 상품명은 자산 변경 가능성이 있으므로 대본에 하드코딩하지 않는다.
8. 메인 화면, 셀프 견적, 내 견적함을 각각 바로 열 수 있게 준비한다.

## 확정 입력 문장

| 순서 | 입력 | 이유 |
|---|---|---|
| 1 | `중학교 3학년 조카가 처음 PC를 맞추려고 합니다. 배틀그라운드가 잘 돌아가는 컴퓨터를 원합니다.` | 조카 상황과 배틀그라운드 용도를 함께 전달하고 예산만 되묻는지 확인한다. |
| 2 | `배그 같은 게임이 잘 돌아갔으면 좋겠어요.` | 조카·게이밍 문맥을 유지한 채 예산 질문을 이어가는지 확인한다. |
| 3 | `최소 견적으로 맞춰줘.` | 숫자 누락으로 취급하지 않고, 누적 게이밍 문맥을 유지한 최소 구성 3안을 요청한다. |
| 4 | `가격이 조금 올라가도 괜찮으니, 배그 화면이 더 부드럽게 보이도록 바꿔줘.` | **부품 이름도 프레임 수치도 모르는 사용자**를 그대로 재현한다. 서버가 성능 패널의 게임·해상도(기본 배그/4K)로 현재 예상 FPS를 조회하고, 그 위의 첫 체감 구간(60→120→165→240)을 목표로 삼아 명시 목표와 **같은 엔진**을 탄다. AI 변경 미리보기의 점수가 준비되면 중앙 비교가 한 번 열리며, 아무 곳이나 눌러 닫은 뒤 상단 비교에서 적용한다. 목표 수치를 직접 말하고 싶으면 `배그 QHD에서 평균 140FPS 이상 나오는 GPU로 바꿔줘.`도 동일하게 동작한다. |
| 5 | `이 변경안 적용해줘.` | 검증된 GPU 변경 미리보기 1건을 현재 견적에 반영한다. |
| 6 | `현재 견적의 호환 문제를 설명해줘.` | 의도적으로 만든 케이스 장착 실패의 실제 Tool 근거를 확인한다. |
| 7 | `호환되는 케이스로 바꿔줘.` | 현재 케이스를 제외하고 전체 구성 Tool 검증을 통과한 단일 복구안을 즉시 반영한다. |

`120FPS 이상 나오는 GPU로 변경해줘`처럼 게임·해상도가 없는 문장도 오류로 처리하지는 않지만, 서버는 근거를 임의로 고르지 않고 게임 또는 해상도를 되묻는다. 또한 현재 PUBG seed에는 1% low가 없으므로 `안정적`이라는 말을 붙이지 않는다. 발표에서는 4번 문장을 그대로 사용한다.

4번의 상대 표현(`더 부드럽게`, `프레임 올려줘`, `FPS 높여줘`, `더 쾌적하게`)은 LLM을 호출하지 않는 fast path다(로컬 실측 15~66ms). 다만 다음은 **일부러 이 경로로 보내지 않으니** 발표 중 섞어 쓰지 않는다 — 증상 신고(`멈춰요`/`튕겨요` → AS 안내), 모니터·주사율(GPU 교체가 답이 아님), 가정형 질문(`바꾸면 얼마나 올라?` → 읽기 전용 시뮬레이션).

## 4분 진행표

| 시간 | 동작과 설명 | 반드시 보이는 결과 |
|---|---|---|
| 0:00~0:20 | 메인 상단의 `AI 검색`을 선택하고 1번 문장을 전송한 뒤 2번 문장으로 용도를 확인한다. | AI가 조카·배틀그라운드용 PC 문맥을 유지하며 예산만 묻는다. 임의 견적 카드는 아직 만들지 않는다. |
| 0:20~0:45 | 3번 문장을 전송한다. | 현재 ACTIVE 자산으로 만든 8개 필수 부품의 추천 카드가 정확히 3개 나온다. 모든 카드는 전체 Tool 검증에서 `FAIL`이 없어야 한다. |
| 0:45~1:05 | 가장 저렴한 카드를 선택해 셀프 견적에 적용한다. | `/self-quote`로 이동하고 8개 부품, 예상가, 종합 1000점, 게임 예상 성능, 호환 통과 상태가 표시된다. 게임 예상 성능은 최초 `배그 / 4K`가 선택되며, 채팅은 `완성 견적이 담겼습니다`와 현재 종합 점수·PUBG 4K 평균 FPS를 같은 화면 수치로 안내한다. |
| 1:05~1:35 | 4번 문장을 전송한다. 중앙에 나타난 종합 점수·전체 예상가·배그 4K FPS 비교를 확인하고 아무 곳이나 눌러 닫는다. 상단 비교와 `교체해 담기`가 남아 있는 것을 보여준 뒤 5번 문장을 전송한다. | 전체 Tool 검증을 통과한 GPU 변경안 1건이 나오며 중앙 강조는 같은 AI 응답에서 한 번만 표시된다. FPS 근거가 늦으면 점수부터 표시되고 같은 카드가 갱신된다. 닫기는 비교 강조만 해제하므로 실제 draft는 아직 바뀌지 않는다. 적용 후 GPU가 실제로 1개만 교체되고 예상가·게임 성능 표시가 갱신된다. 종합 점수는 파워·장착 여유까지 합산하므로 반드시 상승한다고 설명하지 않는다. |
| 1:35~2:00 | 준비한 장착 불가 케이스를 수동 선택한다. | 상단 경고 행에 실제 초과 치수 또는 규격 불일치가 표시되고 구매 안전 게이트가 `FAIL`을 유지한다. `AI에게 설명` 또는 6번 문장으로 같은 원인을 설명할 수 있다. |
| 2:00~2:30 | 7번 문장을 전송한다. | 서버가 기존 케이스와 다른 단일 후보를 전체 구성에 가상 적용해 검증하고 자동 반영한다. 채팅에는 적용 버튼 대신 이전·변경 케이스와 예상가 차이를 보여주는 `변경 전후` 카드가 나오며, 완료 점수·FPS 안내는 문장 순서대로 표시된다. 적용 후 `size`를 포함한 blocking Tool `FAIL`이 0건이 된다. |
| 2:30~3:10 | 현재 견적을 저장하고 `내 견적함`으로 이동한다. 방금 저장한 가성비 견적과 기존 고성능 견적을 비교한다. | 두 견적의 예상가와 종합 1000점 차이가 보인다. CPU·GPU 등 정량 근거가 있는 부품은 실제 성능 데이터로 비교되고, 정량 근거가 없는 항목은 임의 성능 숫자를 만들지 않는다. |
| 3:10~4:00 | 최종 견적을 선택해 저장 견적 상세를 연다. | 선택 부품 목록, 예상가, 종합 성능 정보, Tool 호환 결과가 함께 보인다. 이 화면을 V1의 예상 견적서로 설명한다. |

## 자동 적용 정책

- AI 추천은 먼저 `변경 미리보기`를 만든다. 채팅 응답만으로 quote draft를 몰래 변경하지 않는다.
- 발표에서 말하는 `자동 교체`는 카드 버튼을 누르거나 바로 다음 턴에 `이 변경안 적용해줘`라고 확인한 뒤 일괄 적용되는 동작을 뜻한다.
- 단, 현재 draft에 케이스 `size` FAIL이 있고 사용자가 명령형으로 호환 케이스 복구를 요청한 경우에는 예외다. 서버가 현재 케이스를 제외한 단일 후보를 전체 Tool로 검증한 경우에만 그 요청 자체를 적용 동의로 보고 즉시 반영한다. 또한 점수 설명에서 이어진 `현재 부품이 여유 있게 들어가는 케이스 추천해줘`는 수치화된 장착 여유가 실제로 개선되고 전체 Tool을 통과한 1위 후보가 정확히 한 건일 때만 자동 반영한다. 일반 케이스 추천이나 근거가 불충분한 후보에는 이 예외를 적용하지 않는다.
- AI 적용 완료 안내는 별도 LLM 호출 없이 현재 draft의 Tool 종합 점수와 현재 선택한 게임·해상도의 FPS 근거를 다시 읽는다. 초기 선택은 PUBG 4K이며, 사용자가 직접 바꾼 선택은 유지한다. 적용 시작 시 선택값을 고정해 이전 견적의 placeholder FPS를 재사용하지 않는다. 화면 수치와 다르거나 완료 메시지가 두 번 나오면 발표를 중단한다.
- 직전 미리보기가 0개이거나 2개 이상이면 `적용해줘`가 과거 카드를 임의로 선택해서는 안 된다.
- GPU·케이스 변경 미리보기와 적용 결과에는 각 카테고리 부품이 정확히 1개만 있어야 한다.

## 단계별 불변식

### 최소 견적 3안

- 첫 턴의 조카·`배틀그라운드` 문맥이 둘째·셋째 턴까지 유지된다.
- `최소 견적`은 예산 누락으로 다시 묻지 않는다.
- 카드 수는 3개, 각 카드의 필수 카테고리는 8개다.
- Tool `FAIL` 카드는 제외 후 카드 수만 줄이는 것으로 끝내지 않고, 검증 가능한 다른 구성으로 3안을 채운다.

### 목표 FPS GPU 변경

- 게임, 해상도, 목표 FPS, 평균 또는 1% low 기준이 응답에 명시된다.
- 현재 GPU가 이미 목표를 충족하면 불필요한 변경안을 만들지 않는다.
- 후보는 FPS 근거가 존재하고 현재 전체 구성의 호환성·전력·장착 검증을 통과해야 한다.
- 변경 전에는 draft가 그대로이고, 확인 후 GPU만 교체된다.
- 다른 시나리오에서 목표 GPU가 현재 PSU 권장 정격을 넘는 경우에는 검증을 우회하지 않는다. 서버가 호환되는 PSU를 찾으면 GPU·PSU 연계 변경 미리보기를 반환하고, 찾지 못하면 실제 power Tool 실패 원인과 다음 조치를 안내한다.
- 게임 FPS 상승과 종합 점수 상승은 같은 의미가 아니다. 현재 demo seed에서는 성능 component가 오르지만 600W 파워 여유가 줄어 종합 점수가 소폭 낮아질 수 있다. 발표에서는 `QHD 평균 130FPS → 140FPS`와 가격 변화를 근거로 설명하고, 종합 점수를 성능 단일 지표처럼 말하지 않는다.

### 호환 케이스 복구

- 현재 케이스를 다시 추천하지 않는다.
- 단순 이름·가격 순 추천이 아니라 현재 전체 구성에 후보를 가상 적용해 검증한다.
- blocking Tool `FAIL` 후보는 미리보기와 적용 대상에 포함하지 않는다.
- 현재 케이스 `size` FAIL이 확인되고 검증된 단일 후보가 있을 때만 자동 반영한다. 후보가 없거나 현재 문제가 케이스 장착 실패가 아니면 미리보기·되묻기 정책을 유지한다.
- 복구 후 상단 경고와 BuildGraph Tool 결과가 같은 상태를 보여야 한다.

### 저장·비교·예상 견적서

- 저장 후 원본 quote draft가 바뀌더라도 저장 견적 snapshot은 유지된다.
- 내 견적함 비교는 선택한 두 견적의 예상가와 종합 1000점을 표시한다.
- 저장 견적 상세는 부품 목록, 예상가, 성능 정보, 호환 결과를 모두 표시한다.

## 발표 중단 조건

- 최소 견적 카드가 3개가 아니거나 필수 카테고리가 빠진 경우
- 추천 또는 변경 미리보기에 blocking Tool `FAIL`이 포함된 경우
- GPU 변경안에 PUBG QHD 평균 140FPS 이상 근거가 없거나 전체 Tool 검증을 통과하지 못한 경우
- AI 응답만으로 확인 전에 quote draft가 변경된 경우
- 케이스 복구 후에도 장착 `FAIL`이 남은 경우
- 저장한 견적이 내 견적함 비교 또는 상세에 나타나지 않는 경우

이 중 하나라도 발생하면 대본 문구를 바꾸어 숨기지 말고 자산·Tool·상태 전이 원인을 먼저 확인한다.
