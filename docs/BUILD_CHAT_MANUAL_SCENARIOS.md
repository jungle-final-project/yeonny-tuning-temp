# Build Chat 기능별 수동 테스트 정답지

홈과 셀프 견적의 챗봇은 같은 대화 기능을 사용한다. 아래에서 기능 하나당 대표 입력 하나만 확인하면 된다.

공통 전제:

- 로그인한 상태에서 AI 챗봇을 연다.
- `현재 견적`이라고 적힌 시나리오는 `/self-quote`에서 부품이 담긴 견적을 연 뒤 실행한다.
- 자연어 견적 변경·시뮬레이션은 먼저 변경안 또는 읽기 전용 비교를 만든다. 변경안 바로 다음 턴에 `적용해줘`라고 명시하면 해당 단일 변경안을 적용할 수 있다. 구체 상품을 직접 누른 RAM/SSD 추천 칩과, 현재 견적에 케이스 `size` FAIL이 있고 서버가 단일 복구안을 검증한 `호환되는 케이스로 바꿔줘`는 예외로 즉시 반영된다.

| 확인할 기능 | 시작 위치 | 입력할 문장 | 기대 결과 |
|---|---|---|---|
| 예산 견적 추천 | 홈 또는 셀프 견적 | `300만원 PC 추천해줘` | 예산에 맞는 추천 견적 카드가 나온다. 카드에서 셀프 견적으로 적용할 수 있다. |
| 발표용 QHD 견적 추천 | 홈 또는 셀프 견적 | `200만 원 예산으로 QHD 게임용 PC 추천해줘` | 8개 필수 카테고리가 담긴 1~3개 견적이 175만~225만원 범위로 나온다. Tool FAIL 조합은 표시되지 않는다. |
| 최종 발표용 최소 게이밍 견적 3턴 (핵심 보장) | 홈 AI 검색 | `중학교 3학년 조카가 처음 PC를 맞추려고 합니다. 배틀그라운드가 잘 돌아가는 컴퓨터를 원합니다.` → `배그 같은 게임이 잘 돌아갔으면 좋겠어요.` → `최소 견적으로 맞춰줘.` | 첫 턴부터 조카와 배틀그라운드 용도를 인식해 예산만 묻고, 둘째 턴은 조카·게임 문맥을 유지한다. 셋째 턴은 숫자 예산을 다시 요구하지 않고 8개 필수 카테고리와 전체 Tool 검증을 통과한 최소 구성부터 정확히 3개 카드를 반환한다. 이전 추천 카드나 현재 draft가 요청에 포함돼도 되묻기 캐시로 돌아가지 않는다. 첫 카드를 적용하면 채팅의 같은 완료 메시지가 현재 종합 점수와 최초 선택된 PUBG 4K 평균 FPS로 갱신된다. 이 흐름의 발표 입력·중단 조건은 `DEMO_4MIN_RUNBOOK.md`를 따른다. |
| 명시 부품 조건 견적 | 홈 또는 셀프 견적 | `RTX 5090 넣고 300만원 이하로 맞춰줘` | RTX 5090 조건을 유지한 결과가 나온다. 예산으로 불가능하면 예산 초과 이유가 함께 안내된다. |
| 복수 명시 부품 조건 견적 | 홈 또는 셀프 견적 | `MSI 메인보드와 RTX 5070 Ti GPU를 반드시 같이 넣어 307만원 이하 게임 PC로 구성해줘` | 문장을 제품별 정답으로 하드코딩하지 않고 `gpt-5.4-mini`가 조건을 분류한다. 반환된 모든 카드가 MSI 메인보드와 RTX 5070 Ti를 함께 유지하며, 조건을 지키느라 예산을 넘으면 초과 경고가 표시된다. |
| GPU 필수 용도 예산 역제안 | 홈 또는 셀프 견적 | `30만원짜리 AI용 컴퓨터 추천해줘` | 임의의 저가 조합을 만들지 않는다. AI 용도에는 그래픽카드가 필요하다는 이유와 현재 자산 기준 GPU 포함 최소 구성가를 안내하고, 최소 예산으로 다시 추천하는 칩을 제공한다. |
| 예산 되묻기 | 홈 또는 셀프 견적 | `게임용 PC 필요해` | 바로 임의 견적을 만들지 않고 현재 게이밍 최소 구성가 이상인 `N만원 게이밍 PC 추천해줘` 칩을 보여준다. 칩 자체에 확인한 용도가 포함되어 그대로 눌러도 문맥을 잃지 않는다. |
| 사용자 맥락을 반영한 다중 턴 되묻기 (핵심 보장) | 홈 또는 셀프 견적 | `중3 아들 피시 맞춰줄건데 피시 추천해줘` → `게임용으로 쓸듯` → `400만원 게이밍 PC 추천해줘` | 첫 턴에는 자녀가 사용할 PC라는 명시 맥락과 기본 최소 구성가를 안내하고 예산·용도를 묻는다. 첫 혼합 칩도 현재 기본/GPU 포함 최소가 이상이어야 한다. 둘째 턴에는 `아드님이 사용하실 게이밍 PC`처럼 앞 맥락을 되받고, 이미 받은 게임 용도를 다시 묻지 않은 채 게이밍 최소 구성가 이상의 완전한 예산 칩만 보여준다. 셋째 턴에는 `400만원 예산으로 맞춰볼게요`라는 연결 문장 뒤 누적된 조건의 8개 부품 추천 카드 1~3개를 보여준다. 모든 카드는 350만~450만원 범위이고 Tool FAIL이 없어야 한다. 이 3턴 흐름은 `state-context-01-recipient-usecase-budget` API·웹 상태형 QA의 필수 통과 사례다. 예산을 먼저 답한 경우도 예산을 다시 묻지 않고 용도만 묻는다. |
| 현재 견적 완성 | 셀프 견적, 일부 부품만 담긴 상태 | `지금 견적 기준으로 나머지 부품 채워줘` | 현재 담긴 부품을 기준으로 완성 조합 제안이 나온다. |
| 현재 견적 전체 예산 조정 | 셀프 견적, 목표보다 비싼 견적 | `전체를 200만원으로 알아서 맞춰줘` | 8개 필수 카테고리와 5개 Tool 검증을 갖춘 전체 변경안 카드 1장과, 고액 부품을 개별 조정하는 칩이 함께 나온다. 전체 카드는 target 예산 밴드를 지키며 카드 버튼을 누르거나 바로 다음 턴에 `적용해줘`라고 확인하기 전에는 장바구니가 바뀌지 않는다. |
| 불가능한 전체 예산 역제안 | 셀프 견적, 100만원보다 비싼 견적 | `호환 안되는 거 전부 100만원 내에서 교체해줘` | 100만원 목표를 임의로 올렸다고 말하지 않는다. 현재 ACTIVE 자산에서 Tool 검증을 통과한 최소 완성 조합과 실제 최소가·부족액·`BUDGET_BELOW_MINIMUM`을 보여주고, GPU/CPU/RAM 등 부품별 조정 칩도 함께 제공한다. Tool 검증이 불완전하면 전체 카드는 숨기고 부품별 칩만 남긴다. |
| 직전 변경안 자연어 적용 | 셀프 견적, 전체 또는 단일 변경안 카드가 바로 직전에 1개 표시된 상태 | `적용해줘` 또는 `그걸로 적용해줘` | Build Chat을 다시 호출하지 않고 기존 일괄 적용 API로 직전 카드만 현재 견적에 반영한다. 적용 성공 직후 `종합 점수와 게임 성능을 확인하고 있습니다`가 표시되고, 현재 draft 재검증이 끝나면 같은 메시지가 종합 점수와 최초 선택된 PUBG 4K 평균 FPS로 바뀐다. 사용자가 게임·해상도를 바꿨다면 그 선택을 유지한다. FPS 근거가 없으면 점수만 안내하며 Tool FAIL이면 0점과 상단 경고 확인을 우선한다. 직전 카드가 없거나 여러 개면 과거 카드를 임의 적용하지 않고 카드 선택을 요청한다. |
| AI 적용 후 성능 안내 | 홈 또는 셀프 견적, AI 추천 카드·변경안·RAM/SSD 직접 추가를 실제 적용한 직후 | 카드 적용 또는 `적용해줘` | 수동 부품 선택에는 반응하지 않고 AI가 실제 draft를 바꾼 경우에만 완료 메시지 1건을 남긴다. 완성 조합은 `완성 견적이 담겼습니다`, 단일 변경은 `요청한 변경이 반영되었습니다`로 구분하며 화면과 같은 종합 점수와 현재 선택한 게임·해상도의 FPS만 표시한다. 최초 선택값은 배그/4K다. 이 안내 때문에 Build Chat을 추가 호출하거나 후속 칩을 만들지 않는다. |
| 조건부 단일 부품 추천 | 홈 또는 셀프 견적 | `RTX 5080 그래픽카드 추천해줘` | RTX 5080 조건에 맞는 부품 후보만 나온다. 제조사 조건은 `MSI 메인보드 추천해줘`로 같은 방식으로 확인할 수 있다. |
| 현재 견적 호환 부품 추천 | 셀프 견적, Intel B860/LGA1851 메인보드가 담긴 상태 | `CPU 추천해줘` | 현재 보드와 호환되는 LGA1851 CPU만 문구와 담기 칩에 나온다. AM5 전용 9950X3D/9900X3D처럼 Tool FAIL인 상품은 노출되지 않으며, 호환 후보가 3개 이상이면 FAIL을 제거한 뒤 뒤 후보로 채운 정상 TOP3가 나온다. RAM·SSD 추천은 교체가 아니라 추가 장착 기준으로 빈 슬롯까지 검증한다. |
| 관계형 부품 추천 | 셀프 견적, 메인보드가 담긴 상태 | `현재 메인보드에 맞는 CPU 추천해줘` | 기준 부품인 메인보드를 추천 대상으로 뒤집지 않고 CPU 후보를 반환한다. 현재 소켓과 Tool을 통과한 후보만 나온다. |
| 현재 구성 fit 추천 | 셀프 견적, 부품이 담긴 상태 | `현재 구성에 전력이 충분한 파워 추천해줘` | 숫자 제약이 없어도 현재 구성에 맞는 PSU 추천으로 처리한다. `현재 견적에 2TB SSD 추천해줘`도 전체 PC 견적이 아니라 STORAGE 후보로 이어진다. |
| 대화 조건 기반 부품 조언 | 셀프 견적, 부품이 담긴 상태 | `지금까지 말한 조건 기준으로 현재 견적에 맞는 고성능 파워 제품 추천해줘` | 서버가 현재 draft와 Tool로 검증한 후보만 유지하고, `gpt-5.4-mini`가 앞 조건을 되받아 추천 이유와 첫 후보 확인 질문을 자연스럽게 말한다. 모델이 새 상품·가격·호환성 사실을 만들면 안 되며, 상품 칩을 누르면 해당 카테고리 변경 미리보기로 이어진다. |
| 저장장치 alias 추천 | 홈 또는 셀프 견적 | `M.2 슬롯에 넣을 2테라 추천해줘` | `M.2`를 STORAGE로 인식하고 2TB 조건을 유지한 후보를 반환한다. 메인보드나 전체 PC 추천으로 바뀌지 않는다. |
| 상품 미지정 담기 되묻기 | 홈 또는 셀프 견적 | `램 하나 넣어줘` | 임의의 고가 RAM을 선택하지 않고 `32GB RAM 추천해줘`, `64GB RAM 추천해줘`, `RAM 최저가로 추천해줘` 칩을 보여준다. 짧게 `32GB`라고 답해도 실제 후보 3개로 이어져야 한다. |
| 생성된 추천 칩 왕복 | 홈 또는 셀프 견적, 빈 견적 | `램 하나 넣어줘` → `32GB` → 첫 상품의 `견적에 담아줘` | 되묻기, 조건 후보, 정확 상품 미리보기가 끊기지 않고 이어진다. 마지막 미리보기에는 선택한 RAM 1개만 있고 Tool blocking FAIL이 없어야 한다. |
| 가성비 추천 방향 | 홈 또는 셀프 견적 | `GPU 가성비로 추천해줘` | 최고가 RTX 5090부터 나열하지 않고 가격 대비 벤치마크 근거가 좋은 후보 3개를 보여준다. 메인보드·케이스·쿨러는 근거 없는 성능 점수를 만들지 않고 가격 부담이 낮은 Tool 통과 후보를 보여준다. |
| 추천 후 현재 상품 제외 | 셀프 견적 | RAM 또는 SSD 추천 상품을 실제 추가한 뒤 같은 조건으로 다시 추천 | 방금 담은 exact 상품은 TOP3에 반복되지 않고 다른 Tool 통과 후보로 채워진다. 슬롯이 가득 찼다면 제품을 억지로 추천하지 않고 제거·호환 문제 설명 칩으로 전환한다. |
| RAM/SSD 추천 칩 직접 추가 | 홈 또는 셀프 견적 | `32GB RAM 최저가로 추천해줘` 후 원하는 `견적에 담아줘` 칩 클릭 | 선택한 RAM/SSD가 즉시 현재 견적에 추가된다. 같은 상품은 수량이 1 증가하고, 다른 RAM/SSD는 별도 항목으로 담긴다. 적용 후 채팅은 갱신된 종합 점수와 근거가 있는 FPS를 안내하며, 슬롯 초과 등 Tool FAIL이면 저장은 유지하되 0점과 상단 경고 확인을 우선한다. |
| 견적 부품 변경 미리보기 | 셀프 견적 | `현재 견적 그래픽카드를 더 싼 걸로 바꿔줘` | 변경 전후 금액이 있는 미리보기 카드가 나온다. 적용 버튼을 누르기 전에는 장바구니가 바뀌지 않는다. |
| 발표용 GPU 한 단계 하향 | 셀프 견적 | `GPU를 한 단계 낮은 더 저렴한 제품으로 바꿔줘` | 현재 GPU보다 낮은 class의 호환 가능한 GPU로 변경 미리보기가 즉시 나온다. 가격·예상 성능·호환성 변화가 함께 표시되고 적용 전에는 장바구니가 바뀌지 않는다. |
| 상대 성능 향상 (최종 발표) | 셀프 견적, PUBG FPS 근거가 있는 현재 견적 | `가격이 조금 올라가도 괜찮으니, 배그 화면이 더 부드럽게 보이도록 바꿔줘` | 부품 이름도 프레임 수치도 모르는 사용자를 가정한다. 성능 패널의 게임·해상도(기본 배그/4K)로 현재 예상 평균 FPS를 조회하고 그 위의 첫 체감 구간(60→120→165→240)을 목표로 삼아, 아래 명시 목표와 **같은 엔진**으로 검증된 변경 미리보기 1건을 낸다. ## 상품 상세 확인

상세로 바로 갈지는 **입력이 상품 1건으로 좁혀지는가**로 갈린다. 현재 카탈로그에서 CPU 모델 토큰(`9800X3D` 등)은 상품이 2건, GPU 모델 토큰(`RTX 5080` 등)은 12건 이상이라 **모델명만으로는 상세로 가지 않는다.** 아래 5개를 위에서부터 순서대로 실행한다(2·3번은 1번 결과를 이어서 쓴다).

| 시나리오 | 화면 | 입력 | 통과 기준 |
| --- | --- | --- | --- |
| 후보 2~4건이면 되묻기 | 홈 또는 셀프 견적 | `9800X3D 상세페이지로 이동해` (모델 토큰만) | 화면을 옮기지 않는다. 상품이 여러 개라 골라 달라고 되묻고, 후보마다 `{정확한 상품명} 상세페이지로 이동해` 칩이 나온다. 이 턴에는 예산 PC 추천 칩이 섞이지 않는다. |
| 되묻기 칩 왕복 | 위 되묻기 직후 | 칩 하나를 그대로 클릭 | 칩 문장이 그대로 전송돼 상품이 1건으로 특정되고, 실제로 그 상세(`/parts/{id}`)로 이동하며 챗 패널이 닫혀 도착 화면이 보인다. 견적 추천이나 새 되묻기로 새지 않는다. |
| 풀 상품명 직접 입력 | 홈 또는 셀프 견적 | 위 칩 라벨의 상품명을 그대로 복사해 `{상품명} 상세페이지로 이동해` | 되묻지 않고 곧바로 해당 상품 상세(`/parts/{id}`)로 이동한다. |
| 후보 5건 이상이면 목록 대체 | 홈 또는 셀프 견적 | `RTX 5080 상세페이지로 이동해` | 되묻지 않고 GPU 후보 목록(`/self-quote?category=GPU&q=...`)으로 이동한다. 답변 문구도 '상세페이지' 약속 대신 목록 안내로 교정된다. 도착한 후보 패널에 검색어가 채워져 있고 **목록이 비어 있으면 안 된다**. |
| 취급하지 않는 모델명 | 홈 또는 셀프 견적 | `9700X3D 상세페이지로 이동해` | "상세페이지로 이동한다"고 말하지 않는다. 화면을 옮기지 않고, 취급하는 상품에서 찾지 못했다고 문장으로 알린다(빈 목록 화면으로 떨구지 않는다). |

상품 상세는 위 챗봇 이동 외에, 챗봇이 보여준 부품 후보나 셀프 견적의 부품 카드를 눌러서도 확인할 수 있다.
